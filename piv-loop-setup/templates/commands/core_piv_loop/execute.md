# /piv:execute：执行实施计划

## 待执行计划

阅读计划文件：`$ARGUMENTS`

## 执行说明

### 第一步：阅读并理解

- 仔细阅读完整计划
- 理解所有任务和依赖关系
- 记录验证命令
- 回顾测试策略

### 第二步：按顺序执行任务

对于「步骤化任务」中的每个任务：

#### a. 定位任务

- 识别目标文件和操作
- 阅读相关现有文件

#### b. 实现任务

- 严格按照规范实现
- 保持与代码模式一致
- 包含适当的类型注解和文档
- 在适当位置添加结构化日志

#### c. 随时验证

- 每次更改后检查语法
- 确保导入正确
- 验证类型正确定义

### 第三步：实现测试策略

完成实现后：
- 创建计划中指定的所有测试文件
- 实现所有测试用例
- 遵循概述的测试方法
- 确保覆盖边界情况

### 第四步：运行验证命令

按顺序执行：

```bash
[lint 命令]
[test 命令]
```

如果任何命令失败：
- 修复问题
- 重新运行命令
- 只有通过后才能继续

### 第五步：最终验证

- [ ] 计划中所有任务完成
- [ ] 所有测试创建并通过
- [ ] 所有验证命令通过
- [ ] 代码符合项目约定
- [ ] 文档已添加/更新

## 输出报告

### 已完成任务

- 已完成任务列表
- 创建的文件（带路径）
- 修改的文件（带路径）

### 添加的测试

- 创建的测试文件
- 实现的测试用例
- 测试结果

### 验证结果

```bash
# 每个验证命令的输出
```

### 准备提交

- 确认所有更改完成
- 确认所有验证通过
- 准备好使用 `/commit` 命令

## 注意事项

- 记录计划中未涵盖的问题
- 解释与计划的偏差
- 不要跳过验证步骤
