# /create-prd：创建产品需求文档

根据对话内容生成产品需求文档。

## 输入

对话中的功能讨论。

## 输出

创建 `.claude/PRD.md`，包含：

```markdown
# 产品需求文档：[功能名称]

## 概述
- 状态：草稿 | 待评审 | 已批准
- 创建日期：[日期]
- 负责人：[姓名]

## 问题陈述
我们要解决什么问题？

## 用户故事
1. 作为[用户]，我希望[动作]以便[收益]
2. 作为[用户]，我希望[动作]以便[收益]

## 需求

### 功能需求
| 编号 | 描述 | 优先级 |
|------|------|--------|
| FR-1 | [需求] | 必须 / 应该 / 可以 |

### 非功能需求
- 性能：[规格]
- 安全：[规格]
- 可用性：[规格]

## 设计

### 用户界面
[线框图/模型链接]

### API 设计
```yaml
# 如果适用
/openapi.yaml
```

## 技术说明
- 依赖项：[列表]
- 约束条件：[列表]
- 假设：[列表]

## 成功标准
- [ ] 指标 1
- [ ] 指标 2

## 时间线
- 预估：[X] 天
- 依赖项：[列表]
```

## 后续步骤
- 与利益相关者评审
- 批准需求
- 进入规划阶段
